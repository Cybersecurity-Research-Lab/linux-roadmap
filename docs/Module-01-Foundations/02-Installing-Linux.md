#  Introduction

Now that you understand what Linux is and why it is important, it's time to start using it.

There are several ways to run Linux, each with its own advantages and disadvantages. Choosing the right method depends on your goals, your hardware, and your level of experience.

For this roadmap, we recommend starting with a **virtual machine** because it is safe, flexible, and allows you to experiment without affecting your main operating system.

---

#  Installation Methods

There are four common ways to use Linux.

## 1. Virtual Machine (Recommended)

A virtual machine allows Linux to run inside your current operating system.

### Advantages

* Safe to experiment
* Easy to reset
* No risk to your files
* Perfect for learning

### Disadvantages

* Uses more RAM and CPU
* Slightly lower performance

---

## 2. Dual Boot

Linux and Windows are installed on the same computer.

You choose which operating system to start when the computer boots.

### Advantages

* Full hardware performance
* Separate operating systems

### Disadvantages

* More complex installation
* Risk of partition mistakes

---

## 3. Dedicated Linux Installation

Linux completely replaces the existing operating system.

### Advantages

* Best performance
* Full Linux experience

### Disadvantages

* Existing operating system is removed

---

## 4. Live USB

Linux runs directly from a USB drive without installing it.

### Advantages

* No installation required
* Good for testing distributions

### Disadvantages

* Slower performance
* Changes may not persist

---

#  Why Ubuntu?

Throughout this roadmap we will use **Ubuntu** because it is:

* Beginner friendly
* Well documented
* Stable
* Widely used
* Supported by a large community

The concepts learned in this course also apply to most other Linux distributions.

#  Common Mistakes

### Allocating too little RAM

The virtual machine may become very slow.

---

### Allocating too much RAM

Your host operating system may become unstable.

---

### Forgetting to enable virtualization in the BIOS

Some computers require virtualization (Intel VT-x or AMD-V) to be enabled before virtual machines will work.

---

### Downloading an unofficial ISO

Always download Linux distributions from their official websites.
